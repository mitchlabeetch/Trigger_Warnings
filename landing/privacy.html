Privacy Policy

Last Updated: November 7, 2024

TL;DR: We don't collect, store, or transmit any personal data. Everything happens locally on your device. Your viewing history and preferences stay private.

1. Introduction

Trigger Warnings ("we", "our", or "the extension") is committed to protecting your privacy. This Privacy Policy explains how we handle information in relation to your use of our browser extension.

2. Information We Do NOT Collect

We do not collect, store, or transmit:

  • Personal identification information
  • Email addresses or account credentials
  • Viewing history or watch patterns
  • Video content or screenshots
  • IP addresses or location data
  • Browser fingerprints or tracking identifiers
  • Usage analytics or telemetry

3. Local Data Storage

The extension stores data locally on your device using browser storage APIs:

3.1 User Preferences

  • Profile settings: Your custom profiles, category preferences, and display settings
  • Storage location: Browser's local storage (chrome.storage.local)
  • Access: Only accessible by the extension on your device
  • Deletion: Can be cleared through extension settings or browser data clearing

3.2 Translation Cache

  • Purpose: Cache translated subtitle text to reduce API calls
  • Data stored: Subtitle text and translations (no video content)
  • Retention: Automatically expires after 7 days
  • Storage location: Browser's localStorage

4. Network Requests

The extension makes limited network requests for functionality:

4.1 Warning Database Queries

  • Data sent: Video platform ID only (e.g., "tt1234567" for IMDb)
  • Purpose: Retrieve trigger warnings for the current video
  • Service: Supabase database (self-hosted, no third-party tracking)
  • Privacy: Video IDs are not linked to your identity or viewing history

4.2 Translation API (Optional)

  • Service: MyMemory Translation API (api.mymemory.translated.net)
  • Data sent: Subtitle text only (for non-English content)
  • When used: Only when English subtitles are unavailable
  • Privacy: MyMemory is GDPR compliant and doesn't track users
  • Opt-out: Translation features can be disabled in settings

5. Third-Party Services

5.1 Supabase (Database Hosting)

  • Purpose: Stores community-submitted trigger warnings
  • Data transmitted: Video IDs for queries, warning submissions
  • Privacy: Self-hosted instance, no cross-service tracking
  • Location: EU-based servers (GDPR compliant)

5.2 MyMemory Translation API

  • Purpose: Translates non-English subtitles for trigger detection
  • Data transmitted: Subtitle text only (no video content)
  • Privacy Policy: MyMemory Privacy
  • Free tier: 10,000 words/day, no account required

6. Permissions Justification

The extension requests the following permissions:

6.1 Storage Permission

  • Why needed: Save user preferences, profiles, and translation cache
  • Data stored: Settings, profiles, cached translations (all local)

6.2 Active Tab Permission

  • Why needed: Detect current video and inject warning banners
  • Access: Only on supported streaming platforms

6.3 Host Permissions

  • Domains: Netflix, YouTube, Prime Video, Hulu, Disney+, Max, Peacock
  • Why needed: Access video player to display warnings and analyze subtitles
  • Scope: Limited to video detection and banner injection

7. Data Processing

7.1 Video Analysis

  • Subtitle Analysis: Processed locally in your browser
  • Photosensitivity Detection: Frame analysis happens client-side
  • Video Content: Never uploaded, transmitted, or stored

7.2 User Submissions

When you submit trigger warnings:

  • Required data: Video ID, category, timestamp, description
  • Optional data: Username (can be anonymous)
  • Storage: Stored in community database for other users
  • Deletion: Contact us to remove your submissions

8. Children's Privacy

The extension does not knowingly collect information from children under 13. The service is designed to be used by anyone who can install browser extensions, but we do not target children specifically.

9. Data Security

  • All network requests use HTTPS encryption
  • No authentication tokens or passwords are stored
  • Local data is protected by browser security mechanisms
  • No server-side user accounts or profiles

10. Your Rights

You have the right to:

  • Access: Review all locally stored data in extension settings
  • Deletion: Clear all extension data through settings or browser
  • Portability: Export your profiles and settings (JSON format)
  • Opt-out: Disable specific features (e.g., translation, analytics)

11. Changes to Privacy Policy

We may update this Privacy Policy from time to time. Changes will be posted on this page with an updated "Last Updated" date. Continued use of the extension after changes constitutes acceptance of the updated policy.

12. Open Source

Trigger Warnings is open source software. You can review our code to verify our privacy claims:

  • GitHub Repository: github.com/lightmyfireadmin/triggerwarnings
  • License: MIT License
  • Transparency: All data handling is visible in source code

13. Third-Party Links

The extension may contain links to external websites (e.g., GitHub, documentation). We are not responsible for the privacy practices of these external sites.

14. International Users

The extension can be used worldwide. Database servers are located in the EU (GDPR compliant). Translation services are provided by MyMemory, which operates globally.

15. Do Not Track

The extension does not track users, so Do Not Track (DNT) settings are not applicable. We respect your privacy by default.
